# Daily Cloud Native News Skill

This directory contains an OpenCode skill for automatically generating daily blog posts about cloud native technology developments.

## Files

### 1. `daily-cloud-native-news.md`
The main OpenCode skill file containing:
- Complete instructions for generating news posts
- Search query templates for each category
- Post generation requirements and quality standards
- Output format specifications
- Usage examples

### 2. `daily-cloud-native-news.json`
JSON schema definition of the skill parameters and output format.

### 3. `scripts/generate_daily_news.sh`
Helper script for manual news post generation:
- Creates placeholder posts for specified date/categories
- Updates category pages
- Supports command-line arguments

## Usage with OpenCode

When using the skill with OpenCode, the AI will:

1. Search for news in each category for the specified date
2. Analyze and summarize the findings
3. Generate Jekyll-compatible Markdown posts
4. Return structured output with front matter and content

### Example Invocation

```json
{
  "date": "2026-01-20",
  "categories": ["cloud-native", "tech", "fintech"]
}
```

## Manual Generation

To generate posts manually:

```bash
# Generate for yesterday's news
./scripts/generate_daily_news.sh

# Generate for specific date
./scripts/generate_daily_news.sh 2026-01-20

# Generate for specific categories only
./scripts/generate_daily_news.sh 2026-01-20 cloud-native,tech
```

| Category | Description | Search Focus |
|----------|-------------|--------------|
| Cloud Native | Core cloud native technology | Kubernetes, Docker, Service Mesh, CNCF |
| Tech | Broader technology news | AI/ML, DevOps, Enterprise Software |
| Fintech | Financial technology | Blockchain, Digital Banking, DeFi |

| Military | Defense applications | DOD, Drone Systems, Edge Computing |

## Post Structure

Each generated post includes:

- **Front Matter**: title, date, author, source URL, categories, tags
- **Summary**: Brief overview of key news
- **Key Developments**: Bulleted news items with sources
- **Analysis**: Deep dive into implications and trends
- **Conclusion**: Key takeaways and outlook

## Quality Standards

The skill ensures:
- Real, accessible source URLs from credible sources
- Professional, informative writing
- Original analysis beyond surface summary
- Comprehensive coverage of significant developments

## Integration with CI/CD

The skill can be integrated into automated workflows:

```yaml
# GitHub Actions example
name: Daily News
on:
  schedule:
    - cron: '0 8 * * *'  # Run daily at 8 AM
jobs:
  generate-news:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Generate news
        run: ./scripts/generate_daily_news.sh
      - name: Commit changes
        run: |
          git config user.name "GitHub Actions"
          git add .
          git commit -m "Update daily news $(date +%Y-%m-%d)"
          git push
```

## Notes

- The skill requires web search capability to find current news
- Placeholder posts generated by the shell script need actual content
- Always review and edit generated posts before publishing
- Source URLs should be verified as accessible
