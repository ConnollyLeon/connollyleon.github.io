# 军事应用：Anduril与AWS移动数据中心将边缘计算推向战场前沿

国防云原生技术应用持续深化。Anduril与Amazon Web Services（AWS）联合推出将AWS Outpost集成到Anduril移动数据中心Menace-I的解决方案，首次在实战环境中验证了战术边缘云计算能力。同时，五角大楼正在规划JWCC后续合同"联合云市场"（Unified Cloud Marketplace），标志着军事云战略从单一供应商采购向多层级云生态的重大转变。

## 主要新闻

### Anduril与AWS：将云计算推到战争最前沿

2026年6月30日，Anduril Industries与AWS宣布将AWS Outpost（AWS的本地云计算设备）集成到Anduril的集装箱化指挥和数据中心Menace-I中。这一组合方案只需两名操作人员即可在10分钟内完成部署，可通过卡车、铁路、空运或直升机吊运进行机动。

AWS全球防务管理总监Liz Martin表示："无论是战场情报分析师还是航线操作人员，真正不同的是这种加固的边缘能力。它采取多种形态。"该方案已经在伊朗冲突中的"史诗之怒行动"（Operation Epic Fury）中得到实战验证，Menace-I已部署于陆军、空军、海军陆战队和海军。

AWS同时宣布Anduril为其"国家安全和防务客户的首选边缘提供商"。这一合作的核心价值在于：将云原生计算能力——包括AI推理、数据处理和存储——直接部署到战术边缘，极大缩短了从传感器到决策者的数据传输延迟。

**Source:** [Anduril and AWS Mobile Data Center for Edge Computing](https://www.defenseone.com/business/2026/06/anduril-and-amazons-mobile-data-center-venture-aims-bring-edge-computing-frontlines/414554/)

### 五角大楼JWCC后续：构建三层级"联合云市场"

2026年6月1日，国防信息系统局（DISA）发布了JWCC后续合同——"联合云市场"（Unified Cloud Marketplace, UCM）的绩效工作声明草案。该市场将分为三个层级：第一层级为超大规模云服务商（如AWS、Azure、Google Cloud、Oracle），提供核心基础设施和平台服务；第二层级为"一切即服务"（XaaS）产品，涵盖软件、平台和基础设施即服务能力；第三层级为商业创新者和小企业，提供满足国防部安全要求的云端技术。

这一多层级架构旨在解决军事云领域长期存在的碎片化问题——不同分类级别、不同军种使用不同云环境导致的互操作性差和管理复杂度高。草案特别强调了对AI、战术边缘计算和跨域安全数据共享的支持。

**Source:** [Pentagon's JWCC Follow-On: Cloud Marketplace](https://defensescoop.com/2026/06/01/pentagon-jwcc-ucm-draft-performance-of-work-statement/)

### Pentagon云战略的全球AI维度

根据The AI Chronicle的深度分析，五角大楼正在将云计算从静态基础设施转变为"活的生态系统"，其中AI和边缘计算构成军事力量的"连接组织"。核心挑战不再仅仅是弗吉尼亚州数据中心服务器的算力，而是"在对抗性物流和低带宽环境中数据的可用性"。这一认识推动了从JEDI到JWCC再到UCM的战略演进，其核心目标始终是：在确保数据主权和安全的前提下，实现军事云服务的灵活性和互操作性。

**Source:** [Pentagon's Global AI Cloud Marketplace Vision](https://theaicronicle.com/en/news/geopolitics/pentagon-cloud-marketplace-ai-edge-computing)

## 分析

Anduril与AWS的合作代表了军事边缘计算的重要范式转变。传统上，军事指挥控制系统假设"持续连接"和"中心化基础设施"，但在现代对抗环境中——通信被干扰、带宽严重受限——这些假设迅速失效。Menace-I + AWS Outpost的组合方案通过将Kubernetes原生的云基础设施（AWS Outpost本质上是在本地运行的AWS服务）嵌入到可快速部署的加固集装箱中，实现了"断开环境下的持续计算能力"。

这一技术路径的创新在于：将云原生技术的声明式配置、自愈能力和弹性扩展特性，移植到资源受限、网络不稳定的战术边缘。AWS Outpost提供与AWS云一致的API和管理体验，使军事用户可以在统一的平台上运行从后方数据中心到前线的全部工作负载。对于作战人员而言，这意味着AI推理、情报分析和战场态势感知能力不再受限于通信链路的可用性。

UCM的三层级架构则反映了军事云市场从"寡头垄断"向"多元竞争"的转变。第三层级专门为小企业和创新者开放，这与DISA此前宣布的JWCC-Next扩容策略一致。更多的中小型专业云服务商（如RGS、Defense Unicorns等）将有机会参与国防云生态。这种变化的驱动因素包括：对供应链多样性的需求（避免单一供应商风险）、对专业场景（如边缘计算、分类环境）的定制化需求，以及国会对大型科技公司在国防合同中占比过高的审查。

从更宏观的视角看，军事云原生技术的发展正在引领"战术云原生"（Tactical Cloud Native）这一新兴范式。传统云原生假设稳定的网络和充足的传统计算资源，而战术云原生则需要在DDIL（断开、降级、间歇、低带宽）环境中保持核心功能。这一技术路径的演进不仅对国防领域有重要意义，也将反哺民用边缘计算、偏远地区服务和灾难恢复等场景。

## 结论

Anduril与AWS的合作以及UCM框架的推出共同描绘了国防云原生技术的未来图景：从数据中心到战术边缘，从单一供应商到多元生态，从静态部署到快速机动。对于云原生技术从业者而言，关注eBPF驱动的网络优化、Kubernetes在资源受限环境中的运行优化以及军事级安全合规（如IL5/IL6）能力，将是参与国防云市场的关键。展望未来，随着AI在军事决策中的深度整合，边缘计算和云原生架构的融合将成为现代军事IT基础设施的核心支柱。
