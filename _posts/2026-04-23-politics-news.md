# 政策动态：美国SECURE Data Act欲统一联邦隐私框架

本周政策领域最重要的动态是美国众议院推出SECURE Data Act，这是迄今为止最严肃的联邦消费者隐私立法尝试。同时，欧盟继续推进云主权战略，法国实施敏感数据保护新规，加拿大组织面临CLOUD Act带来的数据主权挑战。

## 主要新闻 (Main News)

### SECURE Data Act：首个重大联邦消费者隐私法案

美国众议院于2026年4月22日推出了《确保和建立消费者统一权利与执法数据法》（SECURE Data Act，HR 8413），这是119届国会首个建立全面消费者保护规则的重要尝试。该法案由众议院能源和商业委员会副主席John Joyce提出，将适用于在美国开展业务或向美国居民提供产品/服务的实体，以及每年处理超过20万名消费者个人数据且年总收入达到2500万美元以上的企业。

**Source:** [Lawmakers seek to override state data privacy laws with new bill](https://www.cnbc.com/2026/04/22/data-privacy-bill-congress-states.html)

### SECURE Data Act详细分析：联邦预emption与执法机制

该法案采用统一联邦标准的预emption模式，将优先于现有的各州消费者隐私法律、数据经纪人注册要求，甚至可能是部门性的州法律。法案为年收入不足2500万美元的企业提供了额外的收入门槛豁免，并允许消费者获取其个人数据的副本以及在可移植和可用格式中导出。该法案还要求对敏感个人数据处理采取选择加入同意制。

**Source:** [SECURE Data Act: Analysis of the new federal privacy bill](https://iapp.org/news/a/secure-data-act-analysis-of-the-new-federal-privacy-bill)

### 欧盟���员会通过战略采购推进云主权

欧盟委员会通过战略采购推进云主权，于2026年4月17日宣布了符合其严格主权标准的云服务提供商名单。委员会开发了云主权框架，将数字主权转化为客观、可衡量的采购标准。该框架通过八个具体目标评估主权性，从战略、法律、运营和环境考虑，到供应链透明度、技术开放性、安全性和合规性。标准设置了主权有效性保证级别（SEAL），从表示完全缺乏主权性的SEAL-0到要求完整欧盟供应链的SEAL-4。

**Source:** [Commission advances cloud sovereignty through strategic procurement](https://commission.europa.eu/news-and-media/news/commission-advances-cloud-sovereignty-through-strategic-procurement-2026-04-17_en)

### 加拿大组织面临CLOUD Act数据主权挑战

数据主权已成为加拿大组织在日益复杂的地缘政治和监管环境中面临的核心问题。2018年美国CLOUD Act赋予美国政府通过美国公司或受美国控制的外国子公司获取数据的权力，即使该数据存储在加拿大且涉及非美国人员。隐私监管机构建议组织评估可能危及其用户个人信息完整性、安全性和机密性的风险，建议进行隐私影响评估以确定数据处理、存储和访问位置。

**Source:** [Data sovereignty in Canada and the CLOUD Act (2026)](https://www.blg.com/en/insights/2026/04/data-sovereignty-and-the-cloud-act-what-canadian-organizations-should-know)

### 2026年美国贸易壁垒报告针对全球数据主权

美国发布的2026年贸易壁垒报告揭示了美国如何将数据主权作为贸易问题来处理。该报告将加拿大旨在应对CLOUD Act域外范围的 sovereign cloud initiative、土耳其对公共部门云计算的全面禁令以及中国对外资云提供商的禁令归为同一类别。这表明美国对合法安全关切和保护主义措施之间没有区分。

**Source:** [The Global Battle for Data Control: How the 2026 U.S. Report on Trade Barriers Targets Data Sovereignty Worldwide](https://www.michaelgeist.ca/2026/04/the-global-battle-for-data-control-how-the-2026-u-s-report-on-trade-barriers-targets-data-sovereignty-worldwide/)

### 法国 Decree No. 2026-272 敏感数据保护法规生效

法国于2026年4月17日实施了第2026-272号法令，这是关于国家行政管理部门处理的敏感数据保护��重要规定。该法令适用于数字健康机构、安全数据访问中心、激进预防资源中心等六个公共利益集团，要求国家信息系统安全局（ANSSI）制定涵盖信息安全管理、人力资源、第三方管理、物理安全、密码学、访问控制等十个领域的安全要求参考框架。

**Source:** [France: Decree No. 2026-272 on sensitive data protection in cloud services provided to state administrations enters into force](https://digitalpolicyalert.org/event/39316-decree-no-2026-272-on-sensitive-data-protection-in-cloud-services-provided-to-state-administrations-enters-into-force)

## 分析 (Analysis)

SECURE Data Act的推出标志着美国联邦隐私立法进入了新阶段。该法案采用联邦预emption模式，这与之前试图建立"地板"而非"天花板"的努力形成对比。这一转变反映出立法者认识到，在20多个州已经实施不同隐私法律的情况下，真正的统一标准需要联邦优先。然而，该法案缺乏民主党的广泛支持，考虑到需要在参议院获得60票才能克服阻挠议事，其通过前景仍不明朗。

欧盟的云主权框架代表了另一种数据治理思路。通过将主权概念量化为可衡量的标准（SEAL级别），欧盟正在为云服务采购建立可操作的评估体系。这种方法可能会影响全球云服务提供商的产品策略，推动它们提供更多符合特定地区主权要求的服务。

CLOUD Act与数据主权的冲突继续困扰着跨境数据处理。加拿大组织面临的两难境地——既需要使用全球云基础设施，又面临数据可能被美国政府访问的风险——反映了许多国家企业的共同困境。2026年美国贸易壁垒报告将加拿大的 sovereign cloud initiative与其他国家的限制措施同等对待，可能会加剧这种紧张关系。

## 结论 (Conclusion)

本周的政策动态表明，全球数据治理正处于关键转折点。美国SECURE Data Act如果通过，将为科技和金融公司创造统一的合规框架，但各州权利组织可能会强烈反对联邦预emption。欧盟的云主权战略继续深化，为组织提供了更清晰的主权云采购路径。对于跨国企业而言，需要建立灵活的数据治理策略，既能满足各地合规要求，又能在数据主权与运营效率之间取得平衡。建议密切关注SECURE Data Act的立法进展，同时评估云主权框架对多云策略的影响。
