# 科技动态：Claude登陆Azure搭载GB300，Google Cloud推科学AI，HP与OpenAI达成合作

全球AI基础设施格局迎来重要变动：Anthropic与微软达成深度合作，Google Cloud进军科学AI领域，NVIDIA和Palantir推动主权AI，同时CI/CD供应链安全曝光重大系统性风险。

## 主要新闻

### Anthropic Claude在Microsoft Foundry全面可用，基于Nvidia GB300 Blackwell

Anthropic的Claude模型现已通过Microsoft Foundry在Azure上全面可用，运行在Nvidia GB300 Blackwell Ultra系统上。这标志着Anthropic与微软、NVIDIA达成三方深度合作：Anthropic承诺购买300亿美元Azure计算容量并额外签约高达1GW的容量，NVIDIA和微软分别承诺向Anthropic投资高达100亿和50亿美元。Azure上提供11个Claude模型（包括Opus 4.8、Sonnet 4.6等），计费通过Claude Consumption Units在Azure Marketplace完成。

**Source:** [Microsoft brings Claude to its own Azure infrastructure, powered by Nvidia GB300 Blackwell](https://www.neowin.net/news/microsoft-brings-claude-to-its-own-azure-infrastructure-powered-by-nvidia-gb300-blackwell/)

### Google Cloud推出科学AI模型和Gemini for Science

Google Cloud宣布将SandboxAQ的"大型定量模型"（LQM）引入云市场，这些AI模型专为药物发现、材料科学和半导体制造等领域训练。研究人员可在Google Cloud上将Gemini用于推理和界面，同时用LQM进行底层科学计算。Google同时推出Gemini for Science工具包，集成AI联合科学家、AlphaEvolve编码代理和NotebookLM等工具，旨在加速科学研究中常规但费力的步骤。

**Source:** [Google Cloud will sell specialist AI models built for science](https://thenextweb.com/news/google-cloud-science-ai-models-sandboxaq)

### Palantir与NVIDIA推出主权AI引擎

Palantir发布基于NVIDIA AI和Nemotron开放模型的"智能引擎"，专为气隙网络和主权环境设计。该解决方案让客户可在完全隔离的环境中运行、定制和持续改进AI，数据和模型权重完全保留在客户手中。Palantir提供了三个工程层面：部署（将基础/定制模型导入气隙和机密网络）、上下文（生产中提示词、工作流结构和模型行为）和模型（基于专有数据和任务成果改变权重本身）。底层是Palantir的AIP、Ontology、Foundry和Apollo堆栈。

**Source:** [Palantir and Nvidia want to change who owns government AI](https://thenewstack.io/palantir-nvidia-sovereign-ai/)

### NVIDIA发布Secure Agent Workspace参考设计

NVIDIA推出Secure Agent Workspace参考设计，为自治AI代理提供安全治理环境。不同于传统在用户设备上执行的AI代理，该设计将执行隔离在托管的虚拟机环境中，通过SSO进行身份验证、默认阻止互联网访问、沙箱化运行时监控，以及关键操作（如合并代码、更改系统状态）需要人工批准。支持Red Hat OpenShift Virtualization和Microsoft Azure部署。

**Source:** [NVIDIA Unveils Secure Agent Workspace for Enterprise AI Governance](https://thecryptopost.io/nvidia-unveils-secure-agent-workspace-for-enterprise-ai-governance/)

### "Cordyceps" CI/CD漏洞影响数百万仓库

安全公司Novee披露了一类系统性的CI/CD漏洞（代号Cordyceps），影响数百万开源软件仓库。这些漏洞存在于GitHub Actions YAML中——低权限工作流可通过不受信任的PR或评论触发，输出最终导向具有维护者权限的高权限工作流。确认受影响的项目包括Microsoft Azure Sentinel、Google AI Agent Development Kit、Apache Doris、Cloudflare Workers SDK和Python Software Foundation的Black。AI辅助编码导致不安全模式在数百万仓库中自动复制。

**Source:** [Exploitable CI/CD Vulnerabilities Expose Millions of Repositories to Hijacking](https://www.securityweek.com/exploitable-ci-cd-vulnerabilities-expose-millions-of-repositories-to-hijacking/)

## 分析

本周科技新闻的核心主线是AI基础设施的"分层重构"。Anthropic与微软/NVIDIA的三方合作代表了AI模型与云计算基础设施的深度耦合——这不是简单的API接入，而是模型训练和推理直接在Azure基础设施上运行，由GB300 Blackwell提供算力。300亿美元的Azure计算承诺意味着Anthropic正在将自己的命运与微软的云基础设施深度绑定。

Google Cloud推出科学AI模型则代表了AI应用从通用向垂直领域的深化。SandboxAQ的大型定量模型填补了语言模型在科学计算领域的空白，Gemini for Science工具包则试图重新定义科研工作流。这一策略也反映了Google在AI云市场的差异化竞争——通过专业化和科学计算能力突围。

Palantir和NVIDIA的主权AI引擎回应了政府和企业对数据主权和模型安全的担忧。在AI API调用模式日益普及的背景下，"拥有AI而非租用AI"的需求正在增长。NVIDIA的Secure Agent Workspace则为AI代理治理提供了可操作的技术参考。

Cordyceps漏洞的披露是一个重要警示：AI加速开发的同时也加速了不安全模式的扩散。数百万仓库自动生成的有缺陷CI/CD工作流表明，安全治理必须跟上AI编码的速度。

## 结论

2026年6月最后一周的科技新闻展现了一个正在分层和专业化发展的AI产业：基础设施层（微软/NVIDIA/Anthropic的合作）、平台层（Google Cloud的科学AI）、应用层（HP-OpenAI合作）和安全治理层（主权AI、安全代理空间）同时取得重大进展。对从业者而言，CI/CD供应链安全的系统性风险需要立即关注——这可能是AI时代最被低估的安全威胁之一。
