# 科技动态：OpenAI GPT-5.5登陆AWS Bedrock、Microsoft发布MAI-Thinking-1推理模型与Cisco推出AgenticOps平台

本周企业AI和云基础设施领域迎来多项重磅发布：OpenAI的GPT-5.5和Codex在Amazon Bedrock上正式可用；Microsoft AI发布MAI-Thinking-1推理模型，以35B活跃参数实现顶尖推理性能；Cisco在Cisco Live上推出AgenticOps平台Cisco Cloud Control；Google Cloud发布GCS MCP Server连接AI代理与非结构化数据；Anyscale在Azure上推出原生集成。

## 主要新闻

### OpenAI GPT-5.5和Codex在Amazon Bedrock上正式可用

Amazon Bedrock宣布OpenAI GPT-5.5、GPT-5.4以及Codex编码代理正式可用。GPT-5.5作为OpenAI最先进的模型，在Bedrock上以与OpenAI第一方定价相同的价格提供，运行在Bedrock的新一代高性能推理引擎上。Bedrock的推理引擎为每个请求提供独立隔离队列和自动容量管理，请求状态被持久化捕获，硬件故障时可从中断点恢复而非重新开始。Codex作为OpenAI的编码代理，每周有超过500万用户使用，现在可通过Bedrock路由推理，支持VS Code、JetBrains、Xcode等IDE集成。Amazon Bedrock Managed Agents（由OpenAI驱动）即将上线，支持生产级代理部署。

**Source:** [OpenAI models and Codex on Amazon Bedrock are now generally available - AWS Blog](https://aws.amazon.com/blogs/machine-learning/openai-models-and-codex-on-amazon-bedrock-are-now-generally-available/)

### Microsoft AI发布MAI-Thinking-1：35B参数开源推理模型

Microsoft AI正式发布MAI-Thinking-1推理模型，这是一个35B活跃参数、约1T总参数的稀疏MoE（混合专家）模型。尽管参数量远小于Claude Opus 4.6等模型，MAI-Thinking-1在SWE-Bench Pro上与后者持平，在AIME 2025上达到97.0%，AIME 2026上达到94.5%。该模型完全基于企业级、经商业许可的干净数据训练，未从第三方模型蒸馏。MAI-Thinking-1支持256K token上下文窗口（可容纳约600页文档）、函数调用和多层指令跟随。模型在Microsoft Foundry上提供私有预览，即将在MAI Playground上公开预览。

**Source:** [Introducing MAI-Thinking-1 - Microsoft AI](https://microsoft.ai/news/introducing-mai-thinking-1/)

### Cisco推出Cisco Cloud Control：面向关键IT基础设施的AgenticOps平台

Cisco在Cisco Live 2026上发布Cisco Cloud Control，这是一个统一平台，让人和AI代理共同管理、监控和防御关键IT基础设施。平台提供单一视图覆盖网络、安全、计算、可观测性和协作，支持Agent Builder和App Builder两大自定义环境。Agent Builder允许用户构建符合自身策略和工作流的AI代理，连接到50多个第三方平台；App Builder支持通过自然语言提示构建应用。Cisco同时宣布Live Protect扩展到更多产品线，可在运行时防御新发现漏洞——无需重启、升级或停机。在安全领域，Cisco承诺在2026年底前覆盖大部分核心产品的量子安全通信能力。

**Source:** [Cisco Unveils Agentic Platform for Operating and Defending Critical IT Infrastructure - Cisco Newsroom](https://newsroom.cisco.com/c/r/newsroom/en/us/a/y2026/m06/cisco-unveils-agentic-platform-for-operating-and-defending-critical-it-infrastructure.html)

### Google Cloud发布GCS MCP Server：让AI代理原生访问非结构化数据

Google Cloud发布Cloud Storage MCP Server，提供Remote（全托管）和Local（自管理）两种模式，使AI代理可通过Model Context Protocol标准化协议访问GCS中的非结构化数据。Remote MCP Server零部署基础设施，支持Google Antigravity和Anthropic Claude等MCP客户端。Local MCP Server开源并提供自定义工具开发能力。Palo Alto Networks的Strata Co-Pilot、Airwallex的AI Assistant和Snap的Job Optimization Agent均基于该服务构建。所有请求通过IAM进行身份验证，审计日志记录每次交互，支持通过Model Armor防御提示注入等攻击。

**Source:** [Build AI agents faster with GCS MCP server - Google Cloud Blog](https://cloud.google.com/blog/topics/developers-practitioners/build-ai-agents-faster-with-gcs-google-cloud-storage-mcp-server)

### Anyscale在Azure上推出原生集成：企业级主权AI平台

Anyscale宣布Anyscale on Azure公开预览，基于AKS和Azure Resource Manager深度集成。该平台允许企业在自己的Azure租户内运行从多模态数据准备到训练和推理的全生命周期AI工作负载。企业数据、训练管道和模型权重始终不离开客户账户边界，通过Microsoft Entra ID策略和RBAC进行治理。Anyscale on Azure支持MACC消耗承诺，客户报告实验速度提升4倍、AI总拥有成本降低高达90%。该平台定位在托管API经济和碎片化基础设施之间提供第三种选择——开源核心、自有基础设施、完整生命周期支持。

**Source:** [Anyscale Launches on Microsoft Azure as a Native Integration - Anyscale Press](https://www.anyscale.com/press/anyscale-on-azure-cost-efficient-and-sovereign-ai-microsoft-build)

## 分析

OpenAI模型登陆Amazon Bedrock代表了AI市场的格局性变化。这不仅仅是模型分销渠道的扩展，更标志着"模型提供商"和"云基础设施提供商"之间关系从竞争走向深度协同。AWS以与OpenAI第一方相同的定价提供GPT-5.5，同时叠加了Bedrock的企业级治理、安全隔离和容量管理能力——这对大型企业来说极具吸引力。Bedrock Managed Agents的即将推出将使得OpenAI的代理能力与AWS的云基础设施深度绑定，进一步提高客户粘性。

Microsoft的MAI-Thinking-1则展示了另一条路径：通过MoE架构实现"小模型、大能力"。35B活跃参数即可匹配Claude Opus 4.6的编码能力，这意味着推理成本可以大幅降低，且能够在更广泛的部署场景中运行。MAI-Thinking-1完全基于自有数据训练的声明也具有战略意义——在模型供应链安全成为焦点的当下，"无蒸馏"意味着更强的知识产权保护和合规确定性。

Cisco Cloud Control的发布表明网络基础设施巨头正在积极拥抱Agentic AI。Cisco将40年的网络运营数据训练为Deep Network Model，与OpenAI Codex集成提供Agent Builder能力——这实际上是在构建一个面向IT运维的"AI代理应用商店"。Quantu-safe通信承诺也反映了行业对"先收后解密"威胁的紧迫感。

## 结论

OpenAI与AWS的深度合作为企业提供了最开放的顶尖模型访问路径；Microsoft MAI-Thinking-1以35B参数证明模型效率比规模更重要；Cisco Cloud Control标志着IT运维全面进入Agentic时代；GCS MCP Server为AI代理与非结构化数据的连接建立了标准化桥梁。AI平台战争已从"谁的模型更好"演变为"谁的生态系统更完整"——企业应根据自身的数据主权需求、现有云投资和AI治理要求，在AWS、Azure和Cisco等平台之间做出战略选择。
