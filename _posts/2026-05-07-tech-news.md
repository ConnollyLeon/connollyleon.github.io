# 科技动态：Anthropic与Google的2000亿美元云交易与ServiceNow知识2026大会

过去24小时内，企业级AI和云服务领域迎来一系列重大公告。Anthropic与Google Cloud签订了价值约2000亿美元的五年代协议，ServiceNow在知识2026大会上发布了AI控制塔扩展和Otto AI助手，谷歌在Cloud Next 2026上推出了Agentic Data Cloud，企业AI竞争格局正以前所未有的速度演进。

## 主要新闻

### Anthropic与Google Cloud签订2000亿美元云服务大单

据The Information援引知情人士消息，AI初创公司Anthropic已承诺在未来五年向Google Cloud支付约2000亿美元云服务费用，这使其成为史上最大的云计算产能购买者之一。该协议使得Anthropic一家就占据了Alphabet近期披露的约40%的收入积压。目前，Anthropic和OpenAI的云服务合同合计占AWS、Microsoft Azure和Google Cloud近2万亿美元收入积压中的约一半。Anthropic此前预计今年将在云计算上花费超过200亿美元，而OpenAI预计将花费约450亿美元。为获得产能，Anthropic还与Broadcom和Google合作开发多个千兆瓦级TPU产能（预计2027年上线），并与CoreWeave和AWS签订了多年协议。这种高度集中的需求模式正在重塑云计算经济——云提供商同时从算力租赁和模型转售中获得高利润收入。

**Source:** [Anthropic strikes massive cloud pact with Google](https://www.digitimes.com/news/a20260506VL215/anthropic-google-google-cloud-revenue-capacity.html)

### ServiceNow知识2026：AI控制塔扩展与Otto AI助手发布

ServiceNow在其年度知识2026大会（拉斯维加斯，5月5-7日）上发布了全面的AI产品组合扩展。核心亮点包括：(1) ServiceNow Otto——集成对话AI、自主工作流和企业搜索的统一AI体验；(2) AI Control Tower扩展——覆盖30+企业集成，提供代理行为的实时可观测性、自动合规控制、AI支出仪表盘和"代理终止开关"；(3) Action Fabric和MCP Server——使外部AI代理（包括Claude、Copilot等）能够安全地触发ServiceNow工作流；(4) Build Agent——扩展到Cursor、Windsurf、Claude Code和GitHub Copilot等开发工具。ServiceNow CEO Bill McDermott将公司定位从"平台之平台"提升为"AI代理之代理"，目标在2030年达到300亿美元订阅收入，其中AI贡献超过30%的年合同价值。

**Source:** [ServiceNow unveils Otto & expands AI Control Tower](https://itbrief.asia/story/servicenow-unveils-otto-expands-ai-control-tower)
**Source:** [ServiceNow wants to be 'AI agent of agents' with Otto platform](https://www.siliconrepublic.com/machines/servicenow-wants-to-be-ai-agent-of-agents-with-otto-platform-and-ai-tools)

### Google Cloud Next 2026：Agentic Data Cloud重新定义数据平台

谷歌在Google Cloud Next 2026大会上发布了Agentic Data Cloud，将原有数据平台重新定义为支持自主AI代理的语义和编排层。关键更新包括：(1) Cross-Cloud Lakehouse——允许企业在AWS和Azure上运行BigQuery和AI能力；(2) Spanner Omni——将全球分布式数据库从Google基础设施解耦，支持隔离环境的向量处理和图形能力；(3) Data Agent Kit——支持Claude Code、Gemini CLI、Codex和VS Code扩展，为开发人员提供构建数据代理的工具。Gartner副总裁分析师Moutusi Sau指出，谷歌的Agentic Data Cloud将数据平台重塑为代理的语义和编排层，解决了因上下文不完善和语义不一致导致的代理失败问题。

**Source:** [Google's Agentic Data Cloud to power 'systems of action'](https://www.computerweekly.com/news/366642734/Googles-Agentic-Data-Cloud-to-power-systems-of-action)

### Atlassian Team '26：开放Teamwork Graph赋能代理化协作

Atlassian在其Team '26大会上宣布开放Teamwork Graph，让Rovo和生态代理能够在企业工具和团队之间安全地搜索、推理和行动。关键进展包括：(1) Rovo已被超过75%的财富500强公司和90%的Atlassian企业云客户使用，月均用户执行超过1400万次Rovo辅助操作；(2) Rovo Studio——提供自然语言驱动的代理和自动化设计工作空间；(3) Max mode——支持将复杂多步骤任务从Rovo Chat中交接给自主代理执行；(4) Jira中@提及代理、Confluence内容重塑、DX AI经验等。Atlassian将这些定位为"AI原生企业"的基础设施，让人类站在工作流的关键前沿，将更多执行交给代理。

**Source:** [Atlassian Opens Up Its Teamwork Graph to Power Agentic Work](https://www.businesswire.com/news/home/20260506005521/)

### IBM Think 2026：watsonx Orchestrate与Concert平台全面升级

IBM在Think 2026大会上宣布了企业AI和混合云管理的最大规模扩展。核心产品包括：(1) watsonx Orchestrate下一代版本——成为多代理时代的代理控制平面，支持来自任何来源的代理与一致策略执行；(2) IBM Concert——一个AI驱动的运营平台，将系统的统一视图与智能响应结合；(3) Sovereign Core GA——在基础设施运行时层面嵌入策略的治理平台；(4) Confluent集成——通过最近收购的Confluent，为AI提供实时数据流。IBM还发布了Concert Secure Coder，将安全管理直接嵌入开发者工作流（Bob和VS Code），利用AI在编码时识别和修复漏洞。

**Source:** [Think 2026: IBM Delivers the Blueprint for the AI Operating Model](https://www.prnewswire.com/news-releases/think-2026-ibm-delivers-the-blueprint-for-the-ai-operating-model-as-the-ai-divide-widens-302762136.html)

## 分析

Anthropic与Google的2000亿美元云交易是AI产业集中化趋势的极佳例证。目前，仅Anthropic和OpenAI两家公司就占全球三大云厂商收入积压的约一半，这种高度集中既是机遇也是风险。对于Google Cloud而言，锁定Anthropic作为锚定客户不仅可以确保长期收入，还能在AI模型竞争（Google Gemini vs Claude）中保持战略平衡。但同时，投资者开始质疑这些巨额支出承诺能否真正实现——它们依赖于激进增长假设，包括两家AI公司预计到本十年末收入增长20-30倍。这反映了AI基础设施军备竞赛的深层悖论：云厂商必须投资于今天的AI领导者才能保住未来市场，但这些领导者自身也存在不确定性。

ServiceNow的知识2026大会和Atlassian的Team '26大会几乎同步举行，两大企业SaaS巨头对AI时代的愿景惊人地一致——都试图成为企业内部AI代理的统一治理和编排层。ServiceNow的"AI代理之代理"和Atlassian的"AI原生企业"讲述的是同一个故事：企业正在从拥有"一个AI助手"转向拥有"数百个AI代理"，而这些代理需要一个中央平台来发现、编排、观察和审计。这解释了双方同时强调Action Fabric/MCP Server和Teamwork Graph的原因——它们都是连接系统的基础设施层。

Agentic Data Cloud是Google对企业AI挑战的独特回应。谷歌的差异化在于它拥有从定制TPU芯片到BigQuery再到Gemini模型的完整技术栈，因此可以提供从底层计算到顶层AI代理的统一平台。Cross-Cloud Lakehouse策略尤其值得关注——承认了多云现实，允许企业在AWS和Azure上使用BigQuery，这既是对企业需求的务实回应，也是一种战略布局：即使计算发生在其他云上，数据治理和分析仍然通过Google平台进行。

IBM Think 2026的公告则展示了一种不同的定位——专注于受监管行业和主权云需求。Sovereign Core的GA发布和Confluent的集成表明，IBM正在将"可解释的AI"和"治理优先"作为核心差异化。在全球数据主权法规日益严格（EU AI Act、DORA、NIS2等）的背景下，IBM的定位可能比亲商更精准。

## 结论

24小时内的五大科技公告揭示了2026年企业AI的关键主题：（1）AI基础设施的集中化达到前所未有的水平，两大AI公司控制着云厂商一半的未来收入；（2）企业软件巨头（ServiceNow、Atlassian、IBM）正在快速将自己重新定位为AI代理的管理平台；（3）数据架构正在被重新设计以支持自主代理——从谷歌的Agentic Data Cloud到Atlassian的Teamwork Graph；（4）治理和合规从"可选"变为"核心功能"——每个平台都在强调审计、安全和可观察性。

对于从业者，建议密切关注：（1）AI基础设施合约对上下游技术栈选择的影响；（2）企业AI代理治理平台的选择标准；（3）数据湖仓架构向代理化演进的路径；（4）AI安全和合规工具与原生产品（如Sovereign Core、AI Control Tower）的整合策略。
