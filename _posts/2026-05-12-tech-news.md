# 科技动态：AI企业级应用爆发——IBM Think 2026、OpenAI-AWS合作与NVIDIA-ServiceNow自治代理

过去一周，企业级AI领域迎来密集发布。IBM在Think 2026大会上推出"AI操作系统模型"，宣布近几十年来最大规模的企业AI和混合云管理能力扩展。OpenAI与AWS大幅扩展合作，将GPT-5.5引入Amazon Bedrock并推出托管代理服务。NVIDIA与ServiceNow联合发布面向企业的自治AI代理Project Arc。Google发布Agentic Data Cloud推动从"智能系统"向"行动系统"的转变。这些发展共同指向一个方向：AI正在从实验阶段全面进入企业生产环境。

## 主要新闻

### IBM Think 2026：全面扩展企业AI与混合云能力

IBM在Think 2026大会上推出了"AI操作系统模型"——一个四层架构，围绕代理（Agents）、数据（Data）、自动化（Automation）和混合基础设施（Hybrid）构建。下一代watsonx Orchestrate进入私有预览，演变为多代理控制平面，支持来自任何来源的代理部署。IBM Sovereign Core正式GA，允许在受控的地理边界环境中部署AI。同时，IBM推出Concert平台（公开预览），从被动监控转向主动协调响应——将应用程序、基础设施和网络信号关联为统一视图，无需替换现有工具。安全方面，Concert Secure Coder在开发者编写代码时实时识别并优先处理风险。

**Source:** [IBM charts AI operating model to move enterprises beyond experimentation](https://siliconangle.com/2026/05/05/ibm-charts-ai-operating-model-move-enterprises-beyond-experimentation/)

### OpenAI与AWS扩大合作：GPT-5.5登陆Amazon Bedrock

OpenAI和AWS宣布大幅扩展合作伙伴关系，将最新的OpenAI模型（包括GPT-5.5）引入Amazon Bedrock，并推出由OpenAI驱动的Amazon Bedrock托管代理和Codex on Amazon Bedrock。这一合作为企业提供了在现有AWS环境（包括安全、合规和采购流程）中构建和运行AI系统的途径。托管代理服务专注于支持保留上下文、执行多步骤工作流、使用工具和执行业务流程的系统。Bedrock托管代理针对OpenAI模型进行了优化，使用OpenAI代理框架解锁前端模型的全部潜力，实现更快执行和更精准推理。

**Source:** [OpenAI & AWS expand tie-up for enterprise AI tools](https://datacentrenews.uk/story/openai-aws-expand-tie-up-for-enterprise-ai-tools)

### NVIDIA和ServiceNow发布企业自治AI代理Project Arc

在ServiceNow Knowledge 2026大会上，NVIDIA和ServiceNow宣布联合推出面向知识工作者（包括开发者、IT团队和管理员）的长期运行、自我演化的自治桌面代理Project Arc。Project Arc使用NVIDIA OpenShell——一个开源安全运行时，用于在沙箱化、策略治理环境中开发和部署自治代理。NVIDIA agent skills使ServiceNow AI Specialists能够在企业工作流中提供针对性能力。平台基于NVIDIA Blackwell架构，相比Hopper提供超过50倍的Token输出每瓦特，每百万Token成本降低近35倍。

**Source:** [NVIDIA and ServiceNow Partner on New Autonomous AI Agents for Enterprises](https://blogs.nvidia.com/blog/servicenow-autonomous-ai-agents-enterprises/)

### Google发布Agentic Data Cloud：从智能系统到行动系统

Google Cloud在Next '26上推出Agentic Data Cloud，将数据平台重新定义为代理的语义和编排层。该架构支持从被动的"智能系统"向自治的"行动系统"转变。发布包括Data Agent Kit（支持Claude Code、Gemini CLI、Codex和VS Code扩展）、Cross-Cloud Lakehouse（允许Google BigQuery和AI能力针对AWS和Azure中的数据运行）、以及Spanner Omni（支持脱离Google基础设施运行，适用于边缘和本地部署）。Gartner分析指出，Agentic Data Cloud直面代理失败的关键原因——不良的数据上下文、不稳定的语义和脆弱的集成。

**Source:** [Google's Agentic Data Cloud to power 'systems of action'](https://www.computerweekly.com/news/366642734/Googles-Agentic-Data-Cloud-to-power-systems-of-action)

## 分析

本周的企业级AI公告呈现一个清晰的共同主线：AI正在从"模型竞赛"转向"运营转型竞赛"。IBM的AI操作系统模型、OpenAI-AWS的托管代理服务和Google的Agentic Data Cloud都指向同一个结论——在2026年，模型本身不再是差异化优势，将AI系统化地集成到企业运营中的能力才是。IBM的Arvind Krishna明确指出"超过70%的企业数据仍位于内部系统中"，这意味着混合云架构（而非单纯公有云或私有云）是企业AI的主要战场。

NVIDIA与ServiceNow的Project Arc代表了企业AI代理的另一个关键趋势：从"对话式AI"向"行动式AI"演进。Project Arc不是回答问题的聊天机器人，而是可以直接访问本地文件系统、终端和应用程序的自治桌面代理。它代表了AI代理从被动辅助角色向主动工作角色的转变。NVIDIA OpenShell的开源策略也值得关注——通过提供安全沙箱运行时，NVIDIA正在将自身定位为企业AI代理的安全标准制定者。

OpenAI与AWS的合作具有更深层的战略意义。OpenAI正在从"自建基础设施"转向"多平台部署"策略，这与其近期与微软关系的逐步松绑一致。通过将模型引入AWS Bedrock，OpenAI获得了进入AWS庞大企业客户群的通道，而AWS则填补了其AI平台中"最先进的推理模型"的空白。Bedrock Managed Agents的推出更是将这个合作推向了"托管服务"层面——企业无需自行搭建代理基础设施即可部署生产级AI代理。

Gartner对Google Agentic Data Cloud的评论值得特别关注："无纪律治理的企业将面临以比代理扩展生产力更快的速度扩展模糊性和不信任的风险。"这意味着企业需要投资于数据语义治理和元数据管理，其重要性不亚于AI工具本身。

## 结论

2026年5月将可能被记住为企业AI从实验走向生产的转折点。IBM的AI操作系统模型、OpenAI-AWS的Bedrock集成、NVIDIA-ServiceNow的自治代理和Google的Agentic Data Cloud从四个不同的角度回答了同一个问题：如何让AI在企业中大规模产生价值。共同答案是：需要运营模型、安全基础设施、数据治理和生态系统的系统化整合，单一模型或工具无法独立完成这一使命。
