# 科技动态：苹果Siri全面重构、微软Foundry企业AI平台与Cisco AgenticOps

过去一周，科技巨头在企业AI和基础设施领域展开密集布局——苹果在WWDC上揭开完全重构的Siri架构，微软在Build 2026推出Foundry企业AI运行时，Cisco Live发布Cloud Control平台。企业级AI正从实验阶段全面进入生产部署阶段。

## 主要新闻

### 苹果WWDC 2026：Siri全面重构，与Google Gemini和NVIDIA深度合作

苹果在WWDC 2026上揭开了完全重构的Siri的面纱。新Siri基于第三代Apple Foundation Models（AFM），采用从零开始重建的架构。核心亮点包括：通过Google Gemini蒸馏技术训练自有模型；Private Cloud Compute扩展到Google云中的NVIDIA GPU；新款AFM Core Advanced模型拥有200亿参数但每次请求仅激活10-40亿。Craig Federighi强调：新Siri运行在苹果自有模型和服务上，Gemini仅用于模型训练蒸馏，不参与用户交互。

**Source:** [Gadgets Now - Inside Apple's Siri Rebuild at WWDC 2026](https://gadgetsnow.indiatimes.com/tech-news/inside-apples-ground-up-siri-rebuild-at-wwdc-2026-with-googles-gemini-and-nvidia/articleshow/131598905.cms)

### Microsoft Build 2026：Foundry定位企业AI基础设施层

微软在Build 2026上宣布Microsoft Foundry的核心基础设施层全面就绪。Foundry Agent Service（预计7月初GA）为每位Agent提供沙箱化运行时环境。新发布的ASSERT框架用于策略驱动的Agent评估，Agent Control Specification（ACS）定义了Agent生命周期的五个安全控制检查点。Toolboxes为Agent提供统一的管理端点，支持MCP协议。微软认为企业AI竞争的关键已从能力转向可靠性和可治理性。

**Source:** [The New Stack - Microsoft Foundry Build 2026](https://thenewstack.io/microsoft-foundry-build-2026-ai-agents/)

### Cisco Live 2026：Cloud Control与AgenticOps时代

Cisco在Cisco Live 2026上发布Cisco Cloud Control平台，这是一套统一的管理层，覆盖网络、安全、计算、可观测性和协作环境。新推出的AgenticOps运营模型允许企业使用AI Agent与人类团队协同自动化基础设施操作。平台包含Agent Builder工具，支持创建定制化AI Agent，并与AWS、ServiceNow、Slack等50多个第三方平台集成。Cisco还计划在2026年底前实现核心产品组合的量子安全通信能力。

**Source:** [ITP.net - Cisco Unveils Cloud Control](https://www.itp.net/ai-automation/cisco-unveils-cloud-control-as-enterprises-enter-the-agentic-ai-era)

### OpenAI Models和Codex登陆AWS

OpenAI宣布其前沿模型和Codex编程Agent已在AWS上全面可用，覆盖Amazon Bedrock和GovCloud区域。通过AWS现有的安全、合规、采购和治理工作流，企业可以在AWS环境中直接调用OpenAI能力。这消除了企业AI采用的最大障碍之一——将前沿AI嵌入到已有基础设施中。

**Source:** [OpenAI - Frontier Models and Codex on AWS](https://openai.com/index/openai-frontier-models-and-codex-are-now-available-on-aws/)

## 分析

本周的三场技术大会（WWDC、Build、Cisco Live）展现了同一趋势的不同侧面：AI正在从"模型竞赛"转向"基础设施竞赛"。

苹果的Siri重构是三者中最激进的——完全抛弃旧架构从零重建。与Google和NVIDIA的合作虽然引人注目，但其核心设计理念是：苹果控制从芯片到模型的每一层。Google负责"教"，NVIDIA负责"跑"，但最终产品是100%苹果自有的。这种"借力但不依赖"的策略可能成为未来科技巨头合作的模板。

微软Foundry的判断值得深思：企业AI的瓶颈不是模型能力，而是可靠性、可治理性和可审计性。ASSERT、ACS和Toolboxes构成了一个完整的生产级AI治理框架，这对于金融、医疗等受监管行业尤为关键。与IBM、KPMG的合作进一步强化了这一信号。

Cisco的Cloud Control和AgenticOps则从基础设施运维的角度切入：大型企业网络和IT基础设施的复杂性与日俱增，AI Agent可以成为运维团队的高效助手，但前提是有严格的安全护栏。Itential FlowAI等第三方产品的参与表明AgenticOps正在形成生态。

## 结论

2026年6月将成为企业AI的一个关键节点。从苹果到微软再到Cisco，科技巨头正在回答同一个问题：如何让AI从演示走向生产？答案各不相同——苹果选择垂直整合，微软选择平台化治理，Cisco选择运维自动化——但共识是清晰的：AI的下一个战场在基础设施层。
